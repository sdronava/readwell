# ReadBook — Implementing Scenario 4: Reader Opens a Book (Local Testing Plan)

## Overview

This document is the detailed implementation plan for **Scenario 4** from `specs/Phase1-MVP.md`:

```
1. Frontend: GET /api/v1/books
   → Gateway queries DB for published books, returns list.
2. User clicks "Read" on a book.
3. Frontend: GET /api/v1/books/{bookId}/metadata
   → Returns title, author, license, cover, total pages.
4. Frontend: GET /api/v1/books/{bookId}/pages/1
   → Returns page 1 structured content blocks (paragraphs, headings, code, images).
5. Frontend renders page from content blocks.
```

**Goal for local testing:** Run the entire read-a-book flow — library view, book open, page navigation, and TTS — entirely on localhost, using a book package already generated by the Document Conversion Service (i.e., the `books/` folder). No S3, no CloudFront, no PostgreSQL, no auth required.

---

## 1. Two-Tier Architecture (Production vs. Local)

The spec defines two distinct serving tiers that **must remain separate**, even locally, because they map to different infrastructure in production:

```
┌──────────────────────────────────────────────────────────────────┐
│                         PRODUCTION                               │
│                                                                  │
│  Frontend (React)                                                │
│      │                                                           │
│      ├─── API calls ──────► Content Gateway API (FastAPI)        │
│      │                          │ reads/writes DB                │
│      │                          │ returns cdnBaseUrl             │
│      │                          ▼                                │
│      └─── content fetches ──► CloudFront CDN                     │
│                                 │ serves books/{bookId}/ from S3 │
└──────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│                     LOCAL TESTING (this plan)                    │
│                                                                  │
│  Frontend (React, :3000)                                         │
│      │                                                           │
│      ├─── API calls ──────► Content Gateway API (:8000)          │
│      │                          │ reads local FS (BOOKS_DIR)     │
│      │                          │ returns cdnBaseUrl             │
│      │                          ▼                                │
│      └─── content fetches ──► Static File Server (:9000)         │
│                                 │ serves published_book/ folder  │
└──────────────────────────────────────────────────────────────────┘
```

Keeping the two tiers separate locally ensures the frontend code is **identical in production and development** — it always follows `cdnBaseUrl` for assets, never talking to the gateway for static content.

## 2. What "Already Generated" Means

The Document Conversion Service produces a book package at `published_book/` (or `books/{bookId}/`):

```
published_book/               ← bookId = "published_book"
├── metadata.json             ← title, author, cover path, totalPages, language, …
├── manifest.json             ← spine order, fileIndex (page → source chapter)
├── chapters.json             ← table of contents [{title, href, depth}]
├── pages/
│   ├── page_001.json         ← blocks[], chapter, section, estimatedReadingTimeSeconds
│   ├── page_002.json
│   └── …  (457 pages for the sample book)
└── assets/
    └── images/
        ├── cover.png
        ├── cover_thumb.webp
        ├── fig_XXX.png
        ├── fig_XXX_400w.webp
        ├── fig_XXX_800w.webp
        └── …
```

The local static file server maps this folder directly onto a URL base. The gateway reads the JSON files from this same folder tree. No conversion, no upload, no DB — just files on disk.

## 3. Component 1 — Local Static File Server (CDN Simulation)

**Port:** `9000`
**Serves:** the parent directory of your book packages (e.g. the project root `/readwell/`)
**Purpose:** mimics CloudFront serving `books/{bookId}/assets/` and `books/{bookId}/pages/`

### Launch command

```bash
# From the project root — serves ALL book packages under published_book/, books/, etc.
python -m http.server 9000
```

Or with `npx`:
```bash
npx serve . --listen 9000 --cors
```

### URL shape

| Resource | URL |
|---|---|
| Cover image | `http://localhost:9000/published_book/assets/images/cover.png` |
| Page 5 JSON | `http://localhost:9000/published_book/pages/page_005.json` |
| Responsive image | `http://localhost:9000/published_book/assets/images/fig_1_1_800w.webp` |

The gateway returns `cdnBaseUrl: "http://localhost:9000/published_book"` for each book, and the frontend constructs all content URLs from this base.

## 4. Component 2 — Content Gateway API

**Port:** `8000`
**Framework:** FastAPI (Python, `uv`)
**Location:** `backend/services/content_gateway/`
**Mode:** `LOCAL_MODE=true` — reads book packages from `BOOKS_DIR` on the local filesystem; no database, no auth

### 4.1 Environment variables (local mode)

| Variable | Example | Description |
|---|---|---|
| `LOCAL_MODE` | `true` | Skip DB and auth; read books from `BOOKS_DIR` |
| `BOOKS_DIR` | `/path/to/readwell` | Root directory containing book package folders |
| `CONTENT_BASE_URL` | `http://localhost:9000` | Base URL of the static file server (CDN simulation) |

### 4.2 Endpoints implemented for Scenario 4

#### `GET /api/v1/books`

In local mode: scan `BOOKS_DIR` for subdirectories that contain a `metadata.json`. Return all of them as "published".

```
Response 200:
{
  "books": [
    {
      "bookId": "published_book",
      "title": "The Logic of Chance, 3rd edition",
      "author": "John Venn",
      "coverUrl": "http://localhost:9000/published_book/assets/images/cover.png",
      "totalPages": 457,
      "description": null,
      "language": "en"
    }
  ]
}
```

#### `GET /api/v1/books/{bookId}/metadata`

Read `{BOOKS_DIR}/{bookId}/metadata.json` and `chapters.json`. Return merged response with `cdnBaseUrl`.

```
Response 200:
{
  "bookId": "published_book",
  "title": "The Logic of Chance, 3rd edition",
  "author": "John Venn",
  "coverUrl": "http://localhost:9000/published_book/assets/images/cover.png",
  "totalPages": 457,
  "language": "en",
  "license": null,
  "cdnBaseUrl": "http://localhost:9000/published_book",
  "tableOfContents": [
    { "title": "Preface", "href": "...", "depth": 0 },
    …
  ]
}
```

#### `GET /api/v1/books/{bookId}/pages/{pageNum}`

Read `{BOOKS_DIR}/{bookId}/pages/page_{pageNum:03d}.json`. Return the file contents directly — the `blocks[]` array is already in the correct format.

```
Response 200:
{
  "bookId": "published_book",
  "pageNum": 1,
  "chapter": "Preface",
  "section": "",
  "estimatedReadingTimeSeconds": 180,
  "blocks": [
    { "type": "heading", "level": 1, "text": "Preface" },
    { "type": "paragraph", "text": "When I first …" },
    …
  ]
}
```

> **Note on assets in blocks:** Image blocks contain `srcset` paths relative to the book package root (e.g. `"assets/images/fig.png"`). The frontend resolves these as `${cdnBaseUrl}/${block.srcset["800w"] || block.srcset.original}`.

#### `GET /api/v1/books/{bookId}/chapters`

Read `{BOOKS_DIR}/{bookId}/chapters.json`. Return the chapters list.

**`local_store.py`** — the core of the local simulation
```python
import json
from pathlib import Path
from content_gateway.config import settings

def list_books() -> list[dict]:
    """Scan BOOKS_DIR for subdirectories containing metadata.json."""
    root = Path(settings.books_dir)
    books = []
    for book_dir in sorted(root.iterdir()):
        meta_file = book_dir / "metadata.json"
        if book_dir.is_dir() and meta_file.exists():
            meta = json.loads(meta_file.read_text())
            books.append(_to_catalog_entry(book_dir.name, meta))
    return books

def get_metadata(book_id: str) -> dict:
    book_dir = Path(settings.books_dir) / book_id
    meta = json.loads((book_dir / "metadata.json").read_text())
    chapters = json.loads((book_dir / "chapters.json").read_text())
    cdn = f"{settings.content_base_url.rstrip('/')}/{book_id}"
    return {**meta, "cdnBaseUrl": cdn,
            "coverUrl": f"{cdn}/{meta.get('cover', '')}",
            "tableOfContents": chapters.get("chapters", [])}

def get_page(book_id: str, page_num: int) -> dict:
    path = Path(settings.books_dir) / book_id / "pages" / f"page_{page_num:03d}.json"
    if not path.exists():
        return None
    return json.loads(path.read_text())

def _to_catalog_entry(book_id: str, meta: dict) -> dict:
    cdn = f"{settings.content_base_url.rstrip('/')}/{book_id}"
    return {
        "bookId": book_id,
        "title": meta.get("title", "Unknown"),
        "author": meta.get("author", "Unknown"),
        "coverUrl": f"{cdn}/{meta.get('cover', '')}",
        "totalPages": meta.get("totalPages", 0),
        "description": meta.get("description"),
        "language": meta.get("language", "en"),
    }
```

**`routers/books.py`**
```python
from fastapi import APIRouter, HTTPException
from content_gateway.config import settings
from content_gateway import local_store, db_store

router = APIRouter(prefix="/api/v1")

def store():
    return local_store if settings.local_mode else db_store

@router.get("/books")
def list_books():
    return {"books": store().list_books()}

@router.get("/books/{book_id}/metadata")
def book_metadata(book_id: str):
    data = store().get_metadata(book_id)
    if data is None:
        raise HTTPException(404, f"Book not found: {book_id}")
    return data

@router.get("/books/{book_id}/chapters")
def book_chapters(book_id: str):
    data = store().get_metadata(book_id)
    return {"bookId": book_id, "chapters": data.get("tableOfContents", [])}

@router.get("/books/{book_id}/pages/{page_num}")
def book_page(book_id: str, page_num: int):
    page = store().get_page(book_id, page_num)
    if page is None:
        raise HTTPException(404, f"Page {page_num} not found in {book_id}")
    return page
```

**`main.py`**

### 5.4 Block rendering

The `Page` component iterates `blocks[]` and dispatches to typed renderers:

```tsx
function Page({ blocks, cdnBaseUrl }: { blocks: Block[]; cdnBaseUrl: string }) {
  return (
    <article>
      {blocks.map((block, i) => (
        <BlockRenderer key={i} block={block} cdnBaseUrl={cdnBaseUrl} />
      ))}
    </article>
  );
}

function BlockRenderer({ block, cdnBaseUrl }) {
  switch (block.type) {
    case "heading":
      const Tag = `h${block.level}` as keyof JSX.IntrinsicElements;
      return <Tag className="font-bold my-4">{block.text}</Tag>;

    case "paragraph":
      return <p className="my-3 leading-7">
        <EmphasisText text={block.text} emphasis={block.emphasis ?? []} />
      </p>;

    case "code":
      return <pre className="bg-gray-100 rounded p-4 overflow-x-auto my-4">
        <code className={block.language ? `language-${block.language}` : ""}>
          {block.text}
        </code>
      </pre>;

    case "list":
      const ListTag = block.ordered ? "ol" : "ul";
      return <ListTag className="list-disc pl-6 my-3">
        {block.items.map((item, j) => <li key={j}>{item}</li>)}
      </ListTag>;

    case "image":
      const src = block.srcset?.["800w"]
        ?? block.srcset?.["400w"]
        ?? block.srcset?.original
        ?? block.filename;
      return <figure className="my-6 text-center">
        <img
          src={`${cdnBaseUrl}/${src}`}
          alt={block.altText}
          srcSet={buildSrcSet(block.srcset, cdnBaseUrl)}
          className="mx-auto max-w-full"
        />
        {block.caption && <figcaption className="text-sm text-gray-500 mt-2">{block.caption}</figcaption>}
      </figure>;

    default:
      return null;
  }
}
```

> **Emphasis rendering:** `EmphasisText` takes the flat `text` string and the `emphasis[]` array (character ranges + style) and wraps the relevant ranges in `<strong>` or `<em>` tags.

### 5.5 TTS (Web Speech API)

The TTS controller:
1. Extracts readable text from `blocks[]` — skips `code` blocks.
2. Calls `window.speechSynthesis.speak(utterance)`.
3. Attaches `boundary` event listener → highlights the current word in the rendered DOM.

```tsx
function useTTS(blocks: Block[]) {
  const speak = () => {
    window.speechSynthesis.cancel();
    const readable = blocks
      .filter(b => b.type === "paragraph" || b.type === "heading")
      .map(b => b.text)
      .join(" ");
    const utterance = new SpeechSynthesisUtterance(readable);
    utterance.addEventListener("boundary", (e) => {
      // highlight word at e.charIndex for e.charLength chars
    });
    window.speechSynthesis.speak(utterance);
  };
  return { speak, stop: () => window.speechSynthesis.cancel() };
}
```
